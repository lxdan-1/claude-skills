# 交互页面生成示例

## 默认输出约定

当用户要求“生成交互页面”“直接给原型”“输出可预览页面”时，默认不只输出方案说明，而是直接创建可运行的 `.html` 文件。

默认应满足：

- 打开 HTML 即可预览
- 核心交互可点击、可输入、可切换
- 页面中预填真实合理的数据
- 至少覆盖一个主路径和一个异常/边界场景
- 如已有对应 PRD，默认与 PRD 文件放在同一目录
- 仅在没有对应 PRD 时，才输出到 `05-项目输出/临时输出/交互原型/`

## 示例1：生成列表页方案

### 输入摘要

```markdown
## 页面目标
支持运营人员统一管理线索，并完成查询、新增、导入和创建任务。

## 适用角色
- 主要角色：运营人员

## 核心场景
- 查询线索
- 新增线索
- 导入线索
- 批量创建任务

## 规则与约束
- 创建任务前必须先勾选线索
- 导入文件需要校验格式和大小
- 不同页签下操作按钮不同
```

### 参考输出

优先直接创建：

- `05-项目输出/阶段产出/线索池PRD/线索池页面原型.html`

文件应至少具备这些能力：

- 支持患者信息、机构信息、违规类型、结算日期筛选
- 支持页签切换：待核实、核实中、已核实
- 支持勾选列表项，未勾选时“创建任务”按钮给出禁用或提示
- 支持点击“新增线索”打开抽屉或弹层
- 支持点击“线索导入”打开导入弹窗
- 列表内置真实业务数据，如患者姓名、机构名称、违规金额、结算日期、线索来源、状态
- 至少准备一组空结果筛选，用于展示空态

建议内置的模拟数据风格：

- 患者姓名：王建国、刘海燕、陈晓敏
- 机构名称：合肥市第一人民医院、蜀山区中医院、庐阳区社区卫生服务中心
- 违规类型：重复收费、超标准收费、项目不匹配
- 线索来源：项目审核、DRG审核、离线导入
- 金额：128.50、2460.00、89.00
- 日期：最近 30 天内的真实格式日期

```markdown
## 页面目标
帮助运营人员在一个页面内完成线索检索、维护、导入和任务创建。

## 页面类型
- 类型：列表页
- 判断依据：核心操作围绕筛选、列表展示、行操作和批量操作展开。

## 适用角色
- 主要角色：运营人员
- 权限差异：待确认是否存在仅查看、可编辑、可创建任务等差异

## 页面入口
- 菜单入口：线索池
- 上游页面：无
- 触发方式：点击左侧菜单进入

## 页面结构
- 顶部区域：页面标题、快捷说明
- 筛选区：患者信息、机构信息、违规类型、结算日期等查询条件
- 操作区：新增线索、线索导入、创建任务
- 页签区：待核实、核实中、已核实
- 列表区：线索列表及行操作
- 底部区域：分页

## 模块说明
### 筛选区
- 作用：支持按多维条件检索线索
- 包含元素：输入框、下拉框、日期选择器、查询按钮、重置按钮
- 关键规则：部分下拉支持多选，重置后恢复默认状态

### 列表区
- 作用：展示线索结果并支持单条或批量操作
- 包含元素：勾选框、列表字段、详情/编辑/删除按钮
- 关键规则：不同页签的列和操作按钮不同

## 核心交互流程
1. 用户进入线索池页面
2. 在筛选区设置条件并点击查询
3. 系统返回列表数据并展示对应页签数量
4. 用户可执行新增、导入、详情、编辑、删除或批量创建任务
5. 系统根据操作结果刷新列表并提示成功或失败原因

## 状态与反馈
- 默认态：展示默认筛选项和第一页数据
- 加载态：查询和导入处理中显示加载状态
- 空态：无查询结果时展示空态说明和返回建议
- 错误态：导入失败或查询失败时提示原因
- 禁用态：未勾选线索时“创建任务”不可执行
- 成功反馈：新增、导入、创建任务成功后提示并刷新列表

## 字段与规则
- 患者信息：
  类型：输入框
  是否必填：否
  校验规则：支持关键字模糊搜索
  联动关系：无
- 违规类型：
  类型：下拉多选
  是否必填：否
  校验规则：仅展示有效字典项
  联动关系：影响列表结果

## 异常与边界处理
- 异常1：导入文件格式不正确时，阻止上传并提示原因
- 异常2：查询失败时保留筛选条件并提示重试
- 边界1：无权限时隐藏或禁用对应按钮
- 待确认项：三个页签的状态定义和切换规则
```

## 示例2：生成表单/抽屉页方案

### 输入摘要

```markdown
## 页面目标
支持运营人员新增一条线索。

## 页面类型
- 倾向：抽屉表单

## 规则与约束
- 附件支持 word/pdf/excel/png/jpg/jpeg
- 最多上传 6 个文件
- 单文件大小 50M 以内
```

### 参考输出

优先直接创建：

- `05-项目输出/阶段产出/线索池PRD/线索新增抽屉原型.html`

文件应至少具备这些能力：

- 点击“新增线索”后打开右侧抽屉
- 可填写文本、选择下拉、上传模拟附件
- 对必填项、字数上限、文件数量与大小做前端校验
- 点击保存后给出成功提示，并将新线索追加到模拟列表或展示回填结果
- 点击关闭时如有未保存改动，给出二次确认

建议内置的模拟数据风格：

- 统筹区：合肥市统筹区、芜湖市统筹区
- 险种类型：职工医保、居民医保
- 机构名称：合肥市第一人民医院
- 违规类型：重复收费、超频次收费、长嘱违规
- 附件名称：病历首页.pdf、处方明细.xlsx、收费清单.jpg

```markdown
## 页面目标
帮助运营人员快速录入线索信息，并完成附件上传与保存。

## 页面类型
- 类型：表单页 + 侧边抽屉
- 判断依据：主要任务是填写字段、上传附件并提交保存。

## 适用角色
- 主要角色：运营人员
- 权限差异：仅具备编辑权限的角色可打开该抽屉并保存

## 页面入口
- 菜单入口：线索池页面
- 上游页面：线索池列表
- 触发方式：点击“新增线索”按钮后，从右侧抽屉打开

## 页面结构
- 顶部区域：抽屉标题、关闭按钮
- 中心区域：基础信息字段、附件上传区域
- 底部区域：取消按钮、保存按钮

## 模块说明
### 基础信息区
- 作用：录入线索基本字段
- 包含元素：统筹区、险种类型、机构名称、违规类型、违规说明等
- 关键规则：部分字段为下拉单选，说明字段有字数上限

### 附件上传区
- 作用：上传线索佐证材料
- 包含元素：上传按钮、文件列表、删除按钮
- 关键规则：限制文件格式、数量和大小

## 核心交互流程
1. 用户点击“新增线索”
2. 系统打开右侧抽屉并展示空白表单
3. 用户填写字段并上传附件
4. 点击保存后系统执行必填和格式校验
5. 校验通过则保存成功并关闭抽屉，否则提示错误

## 状态与反馈
- 默认态：展示空白表单
- 加载态：保存中按钮进入 loading
- 空态：附件区初始为空
- 错误态：字段校验失败或上传失败时展示错误信息
- 禁用态：保存中禁止重复提交
- 成功反馈：保存成功后提示并刷新上游列表

## 字段与规则
- 违规说明：
  类型：文本输入框
  是否必填：待确认
  校验规则：最大 5000 字
  联动关系：无
- 线索附件：
  类型：上传组件
  是否必填：待确认
  校验规则：最多 6 个文件，单文件 50M 以内
  联动关系：上传成功后展示文件列表

## 异常与边界处理
- 异常1：上传超出大小限制时直接拦截
- 异常2：保存失败时保留用户已填内容
- 边界1：关闭抽屉前如有未保存内容，需二次确认
- 待确认项：哪些字段必填，是否支持草稿保存
```

## 使用提示

- 列表页重点拆“筛选区、列表区、操作区、状态反馈”。
- 表单页重点拆“字段、校验、联动、保存反馈”。
- 如果用户已经有 PRD，优先把 PRD 中的业务用例和规则映射成页面模块，不要重新发明结构。
- 如果用户明确要 HTML 原型，默认直接交付文件，不要只停留在 Markdown 说明。
