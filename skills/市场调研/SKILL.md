# 市场调研

## 适用场景

- 需要快速了解某个赛道、产品类型或解决方案。
- 需要对比多个竞品的定位、功能和策略。
- 需要为产品立项、功能规划或方案选型提供依据。

## 执行步骤

1. 明确调研目标：验证机会、对比方案、理解用户，还是辅助立项。
2. 定义调研范围：时间范围、地域范围、产品类型、竞品名单。
3. 建立对比维度：定位、目标用户、核心功能、收费方式、优劣势。
4. 收集证据：官方文档、产品页面、体验记录、行业资料、内部材料。
5. 提炼结论：共性模式、差异化点、进入机会、风险点。
6. 形成建议：给出适合当前业务的方向，而不是只罗列资料。

## 输出要求

优先输出以下结构：

```markdown
## 调研目标

## 调研范围

## 关键发现

## 竞品对比

## 用户洞察

## 机会判断

## 建议动作
```

## 质量检查

- 是否先写清调研目的，再展开材料。
- 是否用统一维度对比，避免横向不可比。
- 是否把“事实”与“判断”分开表达。
- 是否给出结论和建议，而不是只堆素材。
- 是否标注信息来源或证据出处。
