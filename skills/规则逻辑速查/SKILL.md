# 规则逻辑速查

## 适用场景

- 需要从文档或代码中快速解释一条规则如何生效。
- 需要把复杂条件判断整理成可读的逻辑摘要。
- 需要对齐规则口径，减少“各说各话”。

## 执行步骤

1. 锁定规则对象：是哪条规则、哪个节点、哪个模块或哪个场景。
2. 提取关键要素：触发条件、前置条件、判断顺序、执行动作、输出结果。
3. 补充例外情况：兜底逻辑、互斥条件、优先级、异常分支。
4. 标明影响范围：影响哪些角色、页面、数据或流程。
5. 若来源不止一处，指出主依据和冲突点。

## 输出要求

优先输出以下结构：

```markdown
## 规则对象

## 一句话结论

## 触发条件

## 判断逻辑

## 执行动作

## 例外与优先级

## 影响范围

## 依据来源
```

## 质量检查

- 是否用统一术语描述规则。
- 是否把条件、动作、结果分开表达。
- 是否说明例外和优先级，而不只写主逻辑。
- 是否给出来源，便于回溯核对。
- 是否能被产品、研发、测试共同理解。
