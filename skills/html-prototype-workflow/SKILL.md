# HTML 原型生成工作流（政务 / 医疗 / 监管类系统）

## 触发语

用户说以下任意一种时，立即执行本工作流：
- "按方法论帮我生成原型"
- "根据技术要求生成HTML原型"
- "帮我走一遍原型工作流"

## 七步执行流程

### 第一步：需求分析
- 读取技术要求文档，识别**甲方身份**（医保局/卫健委/监管部门）和系统边界
- 区分甲方视角功能 vs 乙方运营功能，标记需删除内容

### 第二步：模块划分
按五层架构归类：**数据底座 / 智能审核 / 监管分析 / 处置运营 / 统计决策**
输出：一级/二级菜单结构

### 第三步：PRD 细化
每个功能点写：**输入 / 处理逻辑 / 输出 / 验收标准**，保存为 `prd.md`

### 第四步：页面映射
每个 FR 对应：页面名 + 入口按钮 + 交互说明，保存为 `页面清单与原型映射.md`

### 第五步：HTML 原型生成
- 左侧 **248px** 固定侧边栏（细分割线 menu-divider，禁用 menu-section 标题）
- 二级菜单用 `data-page` 绑定激活状态，展开 `max-height ≥ 600px`
- 每页：KPI卡（全部用 `<div>`）+ 图表占位 + 数据表格
- 超过4列的表格独占全宽；操作列加 `white-space:nowrap`
- **必须编造真实感数据**（从技术要求提取所有数据实体类型，逐一覆盖）
- **禁止出现**：内测率、准确率(内测)、乙方员工姓名、项目实施进度
- 设计令牌：`--primary:#1658d3; --bg:#eaeff8; --r:10px`

### 第六步：内容审查
- [ ] 是否存在乙方内部数据/指标？→ 删除或替换为甲方业务指标
- [ ] 是否缺少关键监管功能（机构管理、申诉、整改闭环）？→ 补充
- [ ] 规则库/知识库是否给了编辑/发布权限？→ 改为建议/申请

### 第七步：视觉优化
- 配色：冷蓝色系政务风格（`--primary:#1658d3`）
- 侧边栏：品牌LOGO区 + `menu-divider` 细分割线（不用 menu-section）
- sub-item 颜色 `#b8ccf0`（对比度 ≥ 4.5:1）
- 卡片：顶部渐变色条 + hover 上浮效果

## 关键设计原则

1. **甲方视角优先**：所有页面服务于监管人员业务，不展示系统技术细节
2. **数据真实感**：姓名用真实中文名，ID用18位数字，诊断用ICD规范
3. **权责匹配**：甲方只能"查看+建议"，不能直接修改规则库/知识库
4. **PC端优先**：政务系统默认1280px+宽度
