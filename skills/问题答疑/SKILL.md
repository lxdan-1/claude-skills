# 问题答疑

## 适用场景

- 用户提出具体问题，希望快速得到结论和依据。
- 需要结合多份材料解释业务逻辑、页面行为或方案差异。
- 需要在信息不完整时先给出当前判断和待确认项。

## 执行步骤

1. 先识别问题类型：概念解释、流程说明、规则判断、差异对比。
2. 查找相关依据：优先使用现有文档、代码、配置或说明材料。
3. 先给结论，再给原因和依据。
4. 对不确定内容显式标注“已知”“推测”“待确认”。
5. 若问题可延展，补充下一步建议，但不要跑题。

## 输出要求

优先输出以下结构：

```markdown
## 结论

## 依据

## 详细说明

## 待确认项
```

## 质量检查

- 是否先回答用户真正关心的问题。
- 是否给出依据，而不是只给判断。
- 是否区分事实、解释和推测。
- 是否在信息不足时明确说明限制。
- 是否保持简洁，避免把答疑写成大而全文档。
