# PRD写作示例

## 示例1：从需求分析结果生成业务系统类 PRD

### 输入摘要

```markdown
## 一句话需求概述
新增一个线索池管理页面，支持运营人员对线索进行查询、维护、导入和任务创建。

## 需求目标
- 业务目标：提升线索统一管理和线下核查效率。

## 需求类型
- 类型：业务系统类

## 适用角色
- 主要角色：运营人员
- 相关角色：调查员、任务处理人员

## 核心场景
- 查询线索
- 新增线索
- 导入线索
- 批量创建任务

## 主流程
1. 进入线索池页面
2. 执行查询、新增、导入或创建任务
3. 系统保存或处理数据
4. 页面反馈结果
```

### 参考输出骨架

```markdown
# 线索池页面 PRD

## 一、背景与目标
为支持运营人员统一管理待核查线索，需要新增线索池页面，覆盖查询、维护、导入和任务创建等操作。

## 二、范围说明
- 本次包含：线索池页面、基础增删改查、导入、创建任务
- 本次不包含：任务处理后的完整闭环流程

## 三、用户与场景
- 主要角色：运营人员
- 相关角色：调查员
- 核心场景：查询线索、新增线索、导入线索、批量创建任务

## 四、方案说明
### 4.1 页面结构
- 筛选区
- 列表区
- 页签区
- 操作区

### 4.2 业务用例
#### 用例1：查询线索
- 页面入口：线索池页面
- 筛选条件：患者信息、机构信息、违规类型、结算日期等
- 功能按钮：查询、重置

#### 用例2：新增线索
- 表单字段：统筹区、险种类型、机构名称、违规说明、附件等
- 校验规则：必填、字数、格式、附件数量与大小限制

## 五、规则与交互
- 创建任务前必须先勾选线索
- 不同页签下操作按钮不同
- 导入失败时需返回失败原因

## 六、数据与依赖
- 数据来源：线索数据、任务数据、基础字典
- 外部依赖：文件上传、机构数据、用户数据

## 七、验收标准
1. 可完成线索查询、新增、导入、创建任务
2. 页面状态和按钮展示正确
3. 导入与附件校验生效
4. 未选择线索时创建任务给出明确提示

## 八、待确认问题
- 三个页签的状态定义
- 导入模板必填字段
- 角色权限差异

## 交付物建议
- PRD 文件：`05-项目输出/阶段产出/线索池PRD/线索池页面PRD.md`
- 配套原型：`05-项目输出/阶段产出/线索池PRD/线索池页面原型.html`
- 配套原型：`05-项目输出/阶段产出/线索池PRD/线索新增抽屉原型.html`
```

## 示例2：从需求分析结果生成规则逻辑类 PRD

### 输入摘要

```markdown
## 一句话需求概述
新增一条违规判断规则，根据指定条件识别违规记录，并在例外场景下排除误判。

## 需求类型
- 类型：规则逻辑类

## 核心场景
- 命中规则并提示违规
- 满足排除条件，不提示违规

## 规则与约束
- 需要明确触发条件、判断顺序、优先级和排除条件
```

### 参考输出骨架

```markdown
# 某违规判断规则 PRD

## 一、背景与目标
为统一审核口径并减少人工判断成本，新增违规判断规则，对符合条件的数据输出违规提示。

## 二、范围说明
- 本次包含：规则命中逻辑、结果输出
- 本次不包含：规则维护页面调整

## 三、用户与场景
- 主要角色：审核人员
- 场景1：命中规则并提示违规
- 场景2：满足排除条件，不提示违规

## 四、方案说明
### 4.1 规则定义
说明规则识别对象、目标和适用范围。

### 4.2 触发条件
列出规则启动所需条件和输入字段。

### 4.3 处理逻辑
1. 读取输入字段
2. 按顺序执行条件判断
3. 命中后输出违规结果
4. 满足排除条件则终止违规输出

## 五、规则与交互
- 输出违规文案
- 记录命中依据
- 多规则同时命中时按优先级处理

## 六、数据与依赖
- 数据来源：结算数据、规则配置、基础字典
- 依赖：规则引擎、配置管理模块

## 七、验收标准
1. 命中条件时输出正确违规结果
2. 排除条件生效
3. 字段缺失时按预期处理
4. 多规则命中时结果符合优先级

## 八、待确认问题
- 命中文案如何定义
- 字段缺失时处理策略
- 特殊场景是否需要额外排除
```

## 使用提示

- 如果输入还是偏散，先回到 `需求分析` 补结构，再来写 PRD。
- 如果用户已经指定了模板章节，优先保留模板结构，只调整内容填充。
- 如果是强页面型需求，尽量补全字段、按钮、状态和异常反馈。
- 如果是页面型需求，默认把后续 HTML 原型和 PRD 放在同一目录，形成成套交付。
